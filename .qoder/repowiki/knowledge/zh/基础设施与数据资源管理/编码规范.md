- 数据库主键统一采用 BIGINT 类型以支持 Snowflake ID，物理主键使用 BIGSERIAL 自增，业务唯一标识通过 UNIQUE 约束保证。
- 所有核心业务表强制包含 tenant_id 字段并启用 PostgreSQL Row Level Security (RLS) 策略，基于 session 变量 app.current_tenant_id 实现多租户数据隔离。
- 数据库迁移脚本遵循 V{序号}__{描述}.sql 的命名规范，并按功能模块（如 billing, audit_log, agent_marketplace）分阶段演进。
- 知识文档采用 Markdown 格式，并按业务域（biz）、集团职能（group/hr, group/it）进行层级化目录组织，便于 RAG 切片与权限控制。
