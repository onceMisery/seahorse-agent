## 简介
本文件系统性梳理 Seahorse Agent Kernel 模块的领域模型设计，聚焦于 agent 代理模型、chat 聊天模型、conversation 会话模型、ingestion 摄取模型、knowledge 知识库模型、memory 记忆模型、metadata 元数据模型、retrieval 检索模型、**计费模型**、**用户注册模型**、**安全模型**、**知识库增强模型**、**代理市场模型**、**审计日志模型**、**执行步骤模型**等子域的实体、值对象、聚合根与领域服务的定义与职责边界。文档解释各模型之间的关系与约束（继承、组合、依赖），并提供 UML 图与序列图以展示业务流程中的对象交互。最后总结领域模型如何保证业务逻辑的封装性与不变性，并支持业务规则的演进。

## 项目结构
Kernel 模块采用清晰的分层与职责划分：
- config：运行时模式配置
- application：应用服务层，封装业务用例与编排流程
- domain：领域模型与值对象
- feature：可插拔特性接口（L2 扩展）
- plugin：插件化基础设施（扩展注册、激活上下文、特征类型等）
- ports：端口接口（inbound/outbound），作为与外部交互的契约

## 核心组件
本节从领域建模视角，概览 Kernel 子域的关键概念与职责：
- agent 代理模型：封装代理生命周期、工具调用、访问控制与审计等能力
- chat 聊天模型：承载消息、请求、采样与流式回调等聊天相关数据结构
- conversation 会话模型：管理对话历史、附件与状态流转
- ingestion 摄取模型：描述文档摄取任务、上下文与状态机
- knowledge 知识库模型：知识文档、分片、查询与质量度量
- memory 记忆模型：实体别名、记忆项、聚合缓冲与生命周期
- metadata 元数据模型：字典、模式、提取结果与合规治理
- retrieval 检索模型：检索上下文、召回片段与重排序策略
- **billing 计费模型**：订阅计划、支付订单、账单管理与配额控制
- **auth 用户注册模型**：用户注册、验证、免费试用与租户管理
- **security 安全模型**：访问控制决策、资源ACL管理与权限审计
- **knowledge enhancement 知识库增强模型**：版本管理、权限控制、分享链接与访问日志
- **agent marketplace 代理市场模型**：发布审核、订阅管理、评分排行与热度计算
- **audit log 审计日志模型**：访问审计、操作日志与合规追踪
- **execution steps 执行步骤模型**：任务执行跟踪、步骤状态与流程监控

上述模型共同构成 Kernel 的领域模型骨架，支撑上层应用服务完成复杂业务编排。

## 详细组件分析

### Agent 代理模型
- 聚合根：Agent（代理主体），包含工具绑定、访问决策、运行队列与工单状态
- 值对象：AgentToolCall（工具调用）、KernelAgentLoopOptions（循环选项）
- 领域服务：KernelAgentLoop（代理主循环）、KernelAccessDecisionQueryService（访问决策查询）、KernelContextPackBuilderService（上下文包构建）
- 关系与约束：
  - 组合关系：Agent 聚合包含多个运行记录、队列与工单
  - 依赖关系：依赖 ContextPack、AccessDecision、ResourceACL 等端口
  - 不变性：通过只读上下文与不可变工具参数保障执行一致性

## 依赖分析
- 内聚性：每个子域的实体与值对象围绕单一职责边界组织，降低跨域耦合
- 耦合性：应用服务通过端口接口与外部系统解耦；领域模型内部通过组合与依赖表达业务关系
- 外部依赖：向量存储、关键字搜索、对象存储、消息队列、支付网关、邮件服务等通过适配器注入
- 循环依赖：通过端口接口与分层结构避免直接循环依赖

## 性能考虑
- 上下文压缩：通过 MemoryContext 与检索上下文的组合，减少无效信息传输
- 异步编排：摄取与检索通过消息队列与流式回调提升吞吐
- 缓存策略：利用本地/Redis 缓存与分布式锁优化热点访问
- 搜索优化：结合关键字与向量检索，配合重排序策略提升命中质量
- **计费优化**：使用分精度金额避免浮点误差，批量生成账单减少数据库压力
- **安全优化**：默认拒绝策略减少不必要的权限检查开销
- **注册优化**：验证码内存缓存减少数据库查询，异步邮件发送提升用户体验
- **知识库增强优化**：版本快照增量存储，权限缓存预热，分享令牌短时有效
- **代理市场优化**：审核状态机异步处理，评分缓存定期更新，热度计算批处理
- **审计日志优化**：异步写入队列，日志轮转策略，审计事件压缩存储
- **执行步骤优化**：步骤状态持久化，执行指标采样，错误重试指数退避

## 故障排查指南
- 记忆别名状态异常：检查别名状态与实体生命周期是否一致，参考测试用例对状态变更的验证
- 摄取任务卡滞：核对 IngestionContext 与任务状态机推进逻辑
- 检索结果偏差：校验检索上下文与重排序策略的一致性
- **支付订单状态异常**：检查支付回调验签逻辑、幂等性处理和状态机转换条件
- **用户注册失败**：验证验证码有效期、用户名唯一性检查和租户 provision 成功性
- **访问控制异常**：检查 ACL 规则优先级、作用域匹配和审计日志完整性
- **知识库版本回滚失败**：检查快照完整性、版本依赖关系和数据一致性
- **代理发布审核超时**：验证审核状态机、管理员权限和通知机制
- **审计日志丢失**：检查异步写入队列、磁盘空间和日志轮转配置
- **执行步骤中断**：核对步骤状态持久化、重试机制和错误处理策略
