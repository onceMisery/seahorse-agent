## 简介
本指南面向首次接触 Seahorse Agent 的开发者与运维人员，帮助你在最短时间内完成环境准备、项目启动与基础功能验证。项目采用后端微内核 + 前端现代化架构，后端基于 Spring Boot 3.5.7 + Java 17，前端基于 Vite + React。你将学会：
- 安装与配置必需软件（Java 17、Node.js、Docker）
- 初始化数据库与向量数据库
- 启动后端与前端服务
- 体验核心功能（知识库、文档入库、RAG 对话）

## 环境准备
- Java 17+：用于编译与运行后端
- Maven：用于构建与启动后端
- Node.js 与 npm：用于前端开发与构建
- Docker 与 Docker Compose：用于启动 Milvus 与 Pulsar
- Git：用于克隆仓库

## 部署方式
本项目提供多种部署方式，满足不同环境需求：

### 本地开发环境
- 使用默认 Compose 配置启动 PostgreSQL、后端、前端
- 环境变量通过 .env 文件配置，首次运行会自动创建 .env 示例文件
- 前端代理指向后端 9090 端口，便于联调

### Docker 容器化部署
- 后端镜像基于 Eclipse Temurin 17，构建阶段下载依赖并打包
- 前端镜像基于 Node:18-alpine，构建静态资源
- 外部组件通过 Compose 启动，包括 PostgreSQL、Milvus、Pulsar 等

### Kubernetes 集群部署
- 项目提供一键部署脚本，支持最小化与全量部署模式
- 最小化部署：仅启动 PostgreSQL 与 pgvector
- 全量部署：启动 PostgreSQL、Redis、Milvus、Pulsar、Elasticsearch、MinIO 等全套中间件
- 首次构建时间较长，需耐心等待服务健康检查通过

## 常见问题与故障排查
- 启动失败（端口占用）：确认后端 9090、前端 5173、Milvus 19530/9091、Pulsar 6650/8080 未被占用
- 数据库连接异常：检查 JDBC 连接串、用户名与密码，确保 PostgreSQL 已初始化并导入初始数据
- 向量库不可用：确认 Milvus 容器健康状态，查看日志与健康检查端点
- 前端代理无效：检查 Vite 代理配置是否正确指向后端地址
- 全局异常：后端统一返回 code 与 message 字段，便于前端统一处理
