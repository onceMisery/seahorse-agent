## 一、环境准备与访问信息

### 部署方式

推荐使用全量部署以获得完整功能体验：

```bash
# Windows PowerShell
.\deploy.ps1 full

# Linux / macOS
./deploy.sh full
```

### 访问地址

| 地址 | 说明 |
|------|------|
| http://localhost | 前端界面 |
| http://localhost:9090 | 后端 API |
| http://localhost:9090/actuator/health | 健康检查 |
| http://localhost:3001 | Grafana 仪表板（全量） |
| http://localhost:9001 | MinIO 控制台（全量） |
| http://localhost:8000 | Milvus Attu（全量） |
| http://localhost:19090 | Prometheus UI（全量） |

### 默认账号

```
用户名：admin
密码：admin123
```

## 二、登录认证

打开浏览器访问 `http://localhost`，系统会自动跳转到登录页面。

![登录页面](images/guide-login-page.png)

**操作步骤：**

1. 在「用户名」输入框中输入 `admin`
2. 在「密码」输入框中输入 `admin123`
3. 勾选「记住会话」（可选）
4. 点击「进入系统」按钮

登录成功后，系统会自动分配 JWT Token，后续所有 API 请求都需要携带 `Authorization: Bearer <token>` 头。前端会自动管理 Token，无需手动处理。

## 三、工作区仪表板

登录成功后进入工作区首页，这是用户的任务入口。

![工作区仪表板](images/guide-workspace-home.png)

**页面说明：**

- **问候语**：根据时间段显示不同的问候（如"晚上好，admin"）
- **系统状态**：右上角显示当前系统模式（enterprise）和功能通过状态（如 12/13 通过）
- **快捷任务**：提供一键启动的常用任务模板
  - 文档问答：上传文档，智能问答
  - 知识库问答：基于知识库检索回答
  - 运行 Agent：选择 Agent 执行任务
  - 生成 Mermaid 架构图：从 GitHub 仓库自动生成
- **最近任务**：展示最近执行的任务记录
- **最近对话**：展示历史对话列表，点击可继续对话
- **系统能力**：显示当前可用的功能模块状态

## 四、智能对话（Chat）

点击左侧导航栏的「新对话」或访问 `/chat` 进入对话页面。

![聊天页面](images/guide-chat-page.png)

**页面说明：**

- **对话输入**：底部输入框，支持 Enter 发送、Shift+Enter 换行
- **任务类型选择**：可选择「快速回答」等预设任务类型
- **Agent 选择**：可选择已配置的 Agent 进行对话
- **Role Card 选择**：可选择不同的角色卡来定制 AI 的回答风格
- **工具栏**：展开后可配置知识库、模型参数等
- **快速提问**：提供预设的提问模板，一键发送

**使用方式：**

1. 直接在输入框输入问题，按 Enter 发送
2. 选择合适的任务类型和 Agent 获得更精准的回答
3. 系统支持流式响应（SSE），回答会逐步呈现
4. 支持知识库 RAG 检索增强生成

## 五、知识库管理

知识库是 RAG（检索增强生成）的核心。通过管理后台的知识库管理模块可以创建和管理知识库。

![知识库管理](images/guide-knowledge-base.png)

**操作步骤：**

1. 进入管理后台 → 知识库管理
2. 点击「创建知识库」按钮
3. 填写知识库名称、Collection 名称和 Embedding 模型
4. 创建后进入知识库详情页，上传文档
5. 文档上传后触发分块处理
6. 分块完成后即可在对话中使用该知识库进行 RAG 问答

**API 方式创建知识库：**

```bash
curl -X POST http://localhost:9090/knowledge-base \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "demo-kb",
    "collectionName": "demo_kb",
    "embeddingModel": "nomic-embed-text"
  }'
```

## 六、记忆中心

记忆中心允许用户查看和管理 AI 在对话过程中积累的个人记忆。

![记忆中心](images/guide-memory-center.png)

**功能说明：**

- **短期记忆**：当前会话中的对话上下文
- **长期记忆**：经过聚合和压缩后的持久化记忆
- **用户画像**：基于对话自动提取的用户偏好和事实
- **记忆维护**：支持手动触发记忆压缩、清理和别名合并

**验证记忆功能：**

1. 在对话中输入明确的个人事实，如"我的职业是后端工程师，我偏好极简中文回答"
2. 等待聚合窗口（默认 30 秒），或手动触发记忆维护
3. 在记忆中心查看已提取的记忆和画像
4. 开启新会话提问"你记得我的职业和回答偏好吗"，确认记忆召回生效

## 七、管理后台总览

点击工作区右上角的管理入口或访问 `/admin` 进入管理后台。管理后台提供了完整的系统管理和运营功能。

### 7.1 仪表盘

![管理后台仪表盘](images/guide-admin-dashboard.png)

仪表盘是管理后台的首页，提供系统运营的全局视图：

- **核心指标**：活跃用户数、会话数、消息数、会话深度
- **流量概览**：24 小时流量趋势图
- **趋势分析**：会话趋势、活跃用户趋势、响应时间趋势、质量趋势
- **AI 性能**：成功率、平均响应时间、P95 响应时间
- **质量快照**：错误率、无知识率、慢响应率
- **运营洞察**：AI 自动生成的运营趋势分析和建议

### 7.2 知识库管理

详见[第五章](#五知识库管理)。支持知识库的创建、文档上传、分块管理和检索测试。

### 7.3 RAG 评测

![RAG 评测](images/guide-rag-evaluation.png)

RAG 评测模块用于评估知识库的检索和生成质量：

- **检索数据集**：创建和管理评测数据集
- **策略模板**：配置不同的检索策略进行对比
- **版本对比**：对比不同版本知识库的质量变化

### 7.4 链路追踪

![链路追踪](images/guide-rag-traces.png)

RAG Trace 页面展示每次对话的完整链路，包括：

- **检索节点**：知识库检索的命中情况和得分
- **生成节点**：LLM 生成的输入和输出
- **记忆节点**：记忆召回的内容和影响
- **性能指标**：各节点的耗时和 Token 消耗

### 7.5 Agent 管理

![Agent 管理](images/guide-agent-management.png)

Agent 管理模块支持智能体的全生命周期管理：

- **Agent 列表**：查看已创建的所有 Agent
- **创建 Agent**：定义新的 Agent，配置系统提示词、工具绑定和 Skill
- **版本管理**：支持 Agent 的多版本管理和灰度发布
- **Agent 评测**：评估 Agent 的执行效果

### 7.6 Agent 运行管理

![Agent 运行](images/guide-agent-runs.png)

Agent 运行页面展示所有 Agent 的执行记录：

- 运行状态（运行中、已完成、失败）
- 执行步骤和工具调用详情
- 运行耗时和资源消耗

### 7.7 Skill 管理

![Skill 管理](images/guide-skill-management.png)

Skill 是 Agent 可以使用的预定义能力单元：

- 查看系统内置和自定义 Skill
- 创建和管理 Skill 定义
- 将 Skill 绑定到 Agent

### 7.8 工具目录

![工具目录](images/guide-tool-catalog.png)

工具目录管理 Agent 在运行时可调用的**具体工具实例**，是 Agent 绑定和使用的工具注册中心：

- 查看已注册的工具列表（含来源 provider、风险等级）
- 按关键词、风险等级、启用状态筛选工具
- 启用/禁用单个工具，控制 Agent 可用范围
- 管理 MCP Server 连接状态（测试连接、重启、刷新工具列表）
- 点击进入工具详情查看参数配置和调用审计

> **与插件管理的区别**：工具目录面向 **Agent 运行时**，管理 Agent 能使用哪些工具；插件管理面向 **系统基础设施**，管理插件服务的注册和健康状态。

### 7.9 集成连接器

![OpenAPI 连接器](images/guide-openapi-connectors.png)

集成连接器支持对接外部系统：

- **OpenAPI 连接器**：导入 OpenAPI/Swagger 规范，自动生成 API 工具
- **密钥管理**：管理连接器的认证凭据

### 7.10 插件管理

![插件管理](images/guide-plugin-management.png)

插件管理负责系统级 **Plugin 注册表**的生命周期和健康监控：

- 查看所有已注册插件的健康状态（健康/异常计数）
- 启用/禁用插件服务
- 系统级健康检查总览

> **与工具目录的区别**：插件管理关注的是"插件服务本身是否正常运行"，工具目录关注的是"Agent 可以调用哪些工具"。一个插件可能暴露多个工具。

### 7.11 安全治理

![资源 ACL](images/guide-resource-acl.png)

安全治理模块提供企业级的访问控制：

- **资源 ACL**：细粒度的资源访问权限控制
- **访问决策**：查看访问决策的审计记录
- **配额策略**：配置租户和用户的资源使用配额

### 7.12 记忆治理

![记忆治理](images/guide-memory-governance.png)

记忆治理提供记忆系统的健康管理：

- **记忆维护**：手动触发压缩、GC、别名合并
- **记忆质量**：评估记忆的质量和一致性
- **冲突处理**：处理记忆冲突的交互式决策
- **策略配置**：配置记忆聚合和清理策略

### 7.13 审计日志

![审计日志](images/guide-audit-log.png)

审计日志记录系统中所有关键操作：

- 用户登录/登出事件
- 知识库和文档操作
- Agent 配置变更
- 安全策略调整

### 7.14 成本分析

![成本分析](images/guide-cost-analytics.png)

成本分析模块追踪 AI 调用的费用：

- Token 消耗统计
- 按租户、Agent、模型维度的费用分析
- 成本趋势和预算告警

### 7.15 计费管理

![计费管理](images/guide-billing.png)

计费管理支持 SaaS 多租户计费：

- 租户套餐管理
- 用量统计和账单
- 试用额度配置

### 7.16 用户管理

![用户管理](images/guide-user-management.png)

用户管理页面展示系统所有用户：

- 用户列表和角色
- 用户状态管理
- 权限分配

### 7.17 模型配置

![模型配置](images/guide-model-config.png)

模型配置页面展示当前租户的 AI 模型配置：

- Chat 模型（对话生成）
- Embedding 模型（向量编码）
- 模型参数配置

### 7.18 系统诊断

![系统诊断](images/guide-system-readiness.png)

系统诊断页面展示各组件的就绪状态：

- 数据库连接状态
- 向量数据库状态

### 7.19 沙箱

![沙箱](images/guide-sandbox.png)

沙箱提供安全的代码执行环境：

## 十、常见问题排查

### 1. 登录后接口返回"登录已过期"

确认 Token 是否有效：

```bash
curl http://localhost:9090/knowledge-base \
  -H "Authorization: Bearer $TOKEN"
```

### 2. RAG 没有上下文

- 确认文档已完成分块
- 确认 Milvus collection 存在且维度正确
- 在链路追踪页面查看 retrieval 节点

### 3. 记忆没有召回

- 确认对话使用同一个用户
- 确认 `t_short_term_memory` 有数据
- 确认 `t_user_profile_fact` 有 active 画像事实
- 在记忆治理页面触发手动维护

### 4. Milvus 启动慢

Milvus 首次启动需要 60-90 秒，后端有健康检查重试机制，会自动等待。

### 5. 端口冲突

如果端口被占用，修改 `.env` 中的端口映射或停止占用端口的服务。
