一、项目概览

Redis（Remote Dictionary Server）是一个开源的、高性能的内存数据结构存储系统。它可用作数据库、缓存和消息代理。支持字符串、哈希、列表、集合、有序集合、位图、HyperLogLogs、地理空间索引和向量集合等多种数据结构。

关键事实：

  • 许可证：BSD（开源版本）
  • 默认端口：6379
  • 协议：RESP（Redis Serialization Protocol）
  • 主要用途：缓存、会话存储、消息队列、实时分析、向量搜索
  • GitHub 仓库：https://github.com/redis/redis

五、核心逻辑

5.1 命令处理流程

  1. 连接建立：客户端通过 TCP 连接到 Redis 服务器（默认端口 6379）。
  2. 协议解析：Redis 服务器使用 I/O 多路复用（epoll/kqueue）监听客户端连接，接收到数据后通过 RESP 协议解析器将命令转换为内部数据结构。
  3. 命令执行：命令引擎根据命令表查找对应的处理函数，在相应的数据库（db）中执行操作。
  4. 结果返回：执行完成后，结果通过 RESP 协议编码后返回给客户端。

5.2 持久化逻辑

  • RDB：通过 fork() 创建子进程，将内存数据快照写入 RDB 文件。恢复时直接加载 RDB 文件，速度快但可能丢失最后一次快照后的数据。
  • AOF：记录每条写命令到 AOF 文件，支持三种刷盘策略（always/everysec/no）。支持后台重写以压缩文件。数据安全性更高，但文件体积较大。
  • 混合持久化（Redis 4.0+）：结合 RDB 和 AOF 的优势，AOF 重写时先写入 RDB 快照，再追加增量命令。

5.3 复制逻辑

  • 全量同步：从节点发送 SYNC 命令，主节点生成 RDB 文件并发送。
  • 增量同步：主节点将后续命令写入复制积压缓冲区，从节点通过 PSYNC 请求增量命令。
  • 心跳检测：主从节点定期发送 PING 维持连接。
