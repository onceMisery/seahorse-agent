一、项目概览

Redis（Remote Dictionary Server）是一个开源的、基于内存的数据结构存储引擎，被广泛用作缓存、数据库、消息队列和向量搜索引擎。它由 Salvatore Sanfilippo（antirez）于2009年创建，目前由 Redis Ltd.维护。

"For developers, who are building real-time data-driven applications, Redis is the preferred, fastest, and most feature-rich cache, data structure server, and document and vector query engine."

Redis的核心优势在于：极致性能（亚毫秒级延迟）、丰富数据结构、持久化支持、高可用架构和模块扩展能力。

四、流程图

4.1核心命令执行流程

客户端发送命令 →网络层接收 →命令解析 →数据结构操作 →持久化写入 →磁盘存储。Redis通过 aeEvent事件循环高效处理并发请求，单线程执行保证命令原子性。

4.2 Cluster故障转移流程

主节点故障 →哨兵检测 →选举领头哨兵 →投票确认 →从节点提升为新主 →更新集群配置 →恢复服务。

五、核心逻辑

  • 单线程事件循环：aeEvent框架监听文件事件和时间事件，避免锁竞争
  • 数据结构实现：SDS、ziplist、skiplist等精心设计的底层结构
  • 持久化机制：RDB快照、AOF日志、混合持久化
  • 复制与集群：主从异步复制、16384哈希槽分片
