
  混合检索与重排落地设计方案

  ▎ 基于 seahorse-agent 现有微内核架构，不改动 KernelMultiChannelRetrievalEngine 主流程，所有新能力以插件形式接入。

  2. 分阶段落地计划

  P1 — 过滤 + 向量闭环（基础，其余阶段依赖此阶段）

  目标：向量检索真正支持 query embedding 生成和元数据过滤下推。

  2.1 新增核心数据结构

  RetrievalFilter — 位置：kernel/domain/retrieval/

  @Builder
  public record RetrievalFilter(
          String tenantId,           // 生产必填，多租户硬隔离
          String userId,
          List<String> knowledgeBaseIds,
          List<String> documentIds,
          List<String> fileTypes,
          List<String> aclSubjectIds,
          boolean enabledOnly,       // 默认 true
          Map<String, Object> extra
  ) {}

  RetrievalOptions — 位置：kernel/domain/retrieval/

  @Builder
  public record RetrievalOptions(
          int finalTopK,      // 默认 5
          int vectorTopK,     // 默认 finalTopK * 4
          int keywordTopK,    // 默认 finalTopK * 4
          int fusionTopK,     // 默认 finalTopK * 3
          int rerankTopK,     // 默认 finalTopK
          boolean enableVector,
          boolean enableKeyword,
          boolean enableRrf,
          boolean enableRerank,
          String embeddingModel,
          String rerankModel
  ) {
      public static RetrievalOptions defaults(int finalTopK) {
          return RetrievalOptions.builder()
                  .finalTopK(finalTopK).vectorTopK(finalTopK * 4)
                  .keywordTopK(finalTopK * 4).fusionTopK(finalTopK * 3)
                  .rerankTopK(finalTopK).enableVector(true).enableKeyword(false)
                  .enableRrf(true).enableRerank(false).build();
      }
  }

  2.4 过滤表达式模型

  位置：kernel/domain/retrieval/filter/

  public sealed interface MetadataFilterExpr
          permits MetadataFilterExpr.Eq, MetadataFilterExpr.In,
                  MetadataFilterExpr.Range, MetadataFilterExpr.And {

      record Eq(String field, Object value) implements MetadataFilterExpr {}
      record In(String field, List<?> values) implements MetadataFilterExpr {}
      record Range(String field, Object from, Object to) implements MetadataFilterExpr {}
      record And(List<MetadataFilterExpr> children) implements MetadataFilterExpr {}
  }

  public interface MetadataFilterTranslator<T> {
      T translate(MetadataFilterExpr expr);
  }

  RetrievalFilterMapper（kernel 内部工具类）负责将 RetrievalFilter → MetadataFilterExpr：

  public class RetrievalFilterMapper {
      public static MetadataFilterExpr toExpr(RetrievalFilter f) {
          List<MetadataFilterExpr> clauses = new ArrayList<>();
          if (f.tenantId() != null) clauses.add(new Eq("tenant_id", f.tenantId()));
          if (!isEmpty(f.knowledgeBaseIds())) clauses.add(new In("kb_id", f.knowledgeBaseIds()));
          if (!isEmpty(f.documentIds())) clauses.add(new In("doc_id", f.documentIds()));
          if (!isEmpty(f.fileTypes())) clauses.add(new In("file_type", f.fileTypes()));
          if (f.enabledOnly()) clauses.add(new Eq("enabled", true));
          return clauses.size() == 1 ? clauses.get(0) : new And(clauses);
      }
  }

  2.5 向量适配器改造

  Milvus 适配器（seahorse-agent-adapter-vector-milvus）：

  新增 MilvusMetadataFilterTranslator implements MetadataFilterTranslator<String>，输出 Milvus filter 字符串：

  // 示例输出：metadata["tenant_id"] == "t1" and metadata["kb_id"] in ["kb1","kb2"]

  安全要求：字段名白名单校验，字符串值转义，空表达式不设置 filter。

  searchRequest() 改造点：

  String filterStr = filter != null ? milvusTranslator.translate(filter) : "";
  return SearchReq.builder()
          ...
          .filter(filterStr)
          .outputFields(List.of(FIELD_ID, FIELD_CONTENT, FIELD_METADATA))
          .build();

  retrievedChunk() 改造：从 metadata JSON 字段读取 kb_id/doc_id/chunk_index 等并映射到 RetrievedChunk 扩展字段。

  PGVector 适配器（seahorse-agent-adapter-vector-pgvector）：

  新增 PgVectorFilterSqlBuilder，根据 MetadataFilterExpr 动态生成 WHERE 片段（参数化，禁止字符串拼接）。SQL 返回
  metadata 列并映射到 RetrievedChunk。

  2.7 MetadataGuardPostProcessorFeature（order = 50）

  兜底过滤，防止适配器异常导致越权数据穿透：

  public class MetadataGuardPostProcessorFeature implements SearchResultPostProcessorFeature {
      public int order() { return 50; }
      public boolean enabled(SearchContext ctx) { return ctx.getFilter() != null; }

      public List<RetrievedChunk> process(List<RetrievedChunk> chunks,
                                          List<SearchChannelResult> results,
                                          SearchContext ctx) {
          RetrievalFilter f = ctx.getFilter();
          return chunks.stream().filter(c -> passes(c, f)).toList();
      }
  }

  2.12 RrfFusionPostProcessorFeature（order = 100）

  public class RrfFusionPostProcessorFeature implements SearchResultPostProcessorFeature {
      private static final int K = 60;

      public int order() { return 100; }
      public boolean enabled(SearchContext ctx) {
          return ctx.getOptions() != null && ctx.getOptions().enableRrf();
      }

      public List<RetrievedChunk> process(List<RetrievedChunk> chunks,
                                          List<SearchChannelResult> results,
                                          SearchContext ctx) {
          Map<String, RetrievedChunk> merged = new LinkedHashMap<>();
          Map<String, Float> fusionScores = new HashMap<>();

          for (SearchChannelResult result : results) {
              float weight = channelWeight(result.getChannelType());
              List<RetrievedChunk> channelChunks = result.getChunks();
              for (int rank = 0; rank < channelChunks.size(); rank++) {
                  RetrievedChunk c = channelChunks.get(rank);
                  String key = dedupeKey(c);
                  merged.putIfAbsent(key, c);
                  fusionScores.merge(key, weight / (K + rank + 1),
  Float::sum);merged.get(key).getChannelRanks().put(result.getChannelName(), rank + 1);
                  merged.get(key).getChannelScores().put(result.getChannelName(),
                          c.getScore() != null ? c.getScore() : 0f);
              }
          }

          int fusionTopK = ctx.getOptions().fusionTopK();
          return merged.entrySet().stream()
                  .peek(e -> e.getValue().setFusionScore(fusionScores.get(e.getKey())))
                  .map(Map.Entry::getValue)
                  .sorted(Comparator.comparingDouble(c -> -c.getFusionScore()))
                  .limit(fusionTopK)
                  .toList();
      }

      private String dedupeKey(RetrievedChunk c) {
          if (c.getId() != null) return c.getId();
          if (c.getDocId() != null && c.getChunkIndex() != null)
              return c.getDocId() + ":" + c.getChunkIndex();
          return Integer.toHexString(c.getText().strip().hashCode());
      }
  }

  P3 — Reranker 后处理

  2.13 RerankPostProcessorFeature（order = 200）

  public class RerankPostProcessorFeature implements SearchResultPostProcessorFeature {
      private final RerankModelPort rerankModelPort;

      public int order() { return 200; }
      public boolean enabled(SearchContext ctx) {
          RetrievalOptions opts = ctx.getOptions();
          return opts != null && opts.enableRerank()
                  && opts.rerankModel() != null && !opts.rerankModel().isBlank();
      }

      public List<RetrievedChunk> process(List<RetrievedChunk> chunks,
                                          List<SearchChannelResult> results,
                                          SearchContext ctx) {
          RetrievalOptions opts = ctx.getOptions();
          List<RetrievedChunk> candidates = chunks.stream()
                  .limit(opts.fusionTopK()).toList();
          try {
              List<RetrievedChunk> reranked = rerankModelPort.rerank(
                      opts.rerankModel(), ctx.getMainQuestion(), candidates);return reranked.stream()
                      .peek(c -> c.setScore(c.getRerankScore()))
                      .limit(opts.rerankTopK()).toList();
          } catch (Exception ex) {
              LOG.warn("Rerank 失败，降级返回 RRF 结果", ex);
              return candidates.stream().limit(opts.rerankTopK()).toList();
          }
      }
  }

  ┌───────────────────────────────────┬───────┬──────────────────────┐
  │              Feature              │ order │         说明         │
  ├───────────────────────────────────┼───────┼──────────────────────┤
  │ MetadataGuardPostProcessorFeature │ 50    │ 兜底权限过滤         │
  ├───────────────────────────────────┼───────┼──────────────────────┤
  │ RrfFusionPostProcessorFeature     │ 100   │ 多通道 RRF 融合      │
  ├───────────────────────────────────┼───────┼──────────────────────┤
  │ RerankPostProcessorFeature        │ 200   │ 模型重排（可选）     │
  ├───────────────────────────────────┼───────┼──────────────────────┤
  │ FinalTruncatePostProcessorFeature │ 1000  │ 最终截断到 finalTopK │
  └───────────────────────────────────┴───────┴──────────────────────┘

  SearchResultPostProcessorFeature 接口需补充 default int order() { return 0;
  }，KernelMultiChannelRetrievalEngine.executePostProcessors() 在过滤后按 order 排序执行。

  7. 落地顺序建议

  P1 先行（1-2 周）：RetrievalFilter + RetrievalOptions + SearchContext 扩展 + 向量通道 embedding 生成 + Milvus/PGVector
   过滤下推 + MetadataGuardPostProcessorFeature。这是后续所有阶段的基础，且不引入新依赖。

   + Outbox 异步索引同步。
